## 带外和带内(Out-of-Band Deployment)
所谓的"带"从中文上不好理解，但是学通信的朋友看到Band就清楚了，表明的是通信信道。
带内指的就是直接在通信信道内的，可能是整个信道的某一个节点，一般就是跑业务的，更具体一点来说，所有的业务流量都跑在带内的网卡上。而带外的信道，就不跑业务，往往都是用来做管理口的。

软件运行在厂商的云端平台上，用户只需要通过浏览器或客户端 登录即可使用，不需要额外安装硬件。

## x86,arm,amd
- x86-64:它指的是一种 CPU 指令集架构（ISA），指的是 Intel 在1978年发布的 8086 处理器架构,主要被 Intel 和 AMD 用在 个人电脑、服务器 上。
- AMD: 是 CPU/GPU 厂商, AMD 做的桌面/服务器 CPU，大多数是基于 x86 架构（比如 Ryzen、EPYC）。
- ARM: ARM 是英国公司 Arm Ltd. 设计的一种 精简指令集架构（RISC），他只负责设置架构，不自己生成CPU。ARM一般都是在嵌入式，MAC系统上用的。

x86 和 ARM 是两种不同的 CPU 指令集架构，而 Intel 和 AMD 是主要的 CPU 制造商，其中 Intel 和 AMD 的桌面与服务器 CPU 主要采用 x86 架构，而 ARM 架构则广泛应用于移动设备、嵌入式系统以及近年来的 PC 和服务器领域。

## 内存泄露
内存泄露是指程序在运行过程中分配了内存但没有正确释放，导致这些内存无法被垃圾回收器或其他机制回收，最终造成内存占用持续增加。

比如在C++里面程序使用 malloc、new 等分配内存，但忘记调用 free、delete 或析构函数释放。

内存泄露会导致内存占用增加，系统 Swap（虚拟内存）使用增多，I/O 操作变慢，CPU 利用率上升。严重时会触发 OOM（Out of Memory）杀进程，程序强制退出；或整个系统内存耗尽，导致内核 panic。

大部分编程语言和OS都有相应的内存回收机制，内存泄露也可以通过定时重启OS来临时解决。

## 运营商侧刷新域名
如果域名的负责人修改了域名对应的记录，比如说IP地址。

正常来说如果用户走迭代查询，会最后从权威服务器上拿到最新的IP地址。但由于各个运营商的DNS服务器上都设置缓存机制，用户使用运营商的DNS服务器进行解析，可能会命中之前的缓存，导致拿到旧地址。 运营商侧刷新，就是主动去刷他们的DNS服务器上的缓存

运营商刷新都是按次收费的，均价是1W多一次


## 板卡和千兆电口卡
“千兆电口卡”通常指 千兆以太网网卡（Gigabit Ethernet NIC），并且接口类型为电口（RJ45 接口），也就是使用网线（双绞线）的网口卡。

- “板卡”是一个更 宽泛 的术语，常用于通信、服务器、交换机等领域，指任何可以插入主机或机箱、实现特定功能的 电路板模块。

|板卡类型|作用|
|--------|---|
|网卡|提供网络接口（电口/光口）|
|接口板 / 扩展板|增加串口、USB、CAN 等接口|
|业务板|在交换机/路由器中承担转发、处理功能|

## KVM remote是个什么玩意？

KVM(Keyboard + Video + Mouse)
这是在服务器带外管理（out-of-band management）中最核心、最强大的功能之一，通过网络，让你像坐在服务器机房前一样操作一台（通常是物理）服务器

为什么常见的有 Java 和 HTML5 两种?

[是一项技术的新老版本，以前用Java实现起来更方便，但现在html5更好，还不用下任何插件，点击就能用]

Java 版 → 老技术、老设备常用，现在用起来很麻烦（安全限制、Java 版本冲突、浏览器不支持）
HTML5 版 → 新标准、推荐方式，几乎所有 2018 年后出的服务器 BMC 都默认或只提供 HTML5


## 运营商的 ASN 值是啥？

AS就是计算机网路里面学的Autonomous System = 自治系统

中国电信、中国联通、移动、阿里云、腾讯云、Google、Cloudflare、AWS……这些大公司/运营商各自就是一个或多个 AS

一个 AS 通常包含：

大量公网 IP 地址段
一堆按照统一策略管理的路由器

ASN = Autonomous System Number = 自治系统编号（也叫 AS 号）
就是给每个 AS 发的“身份证号”或“车牌号”，全球唯一。

写法通常是 AS + 数字，比如：
AS4134 中国电信骨干网
AS4837 中国联通骨干网

## IOPS
IOPS: 每秒输入/输出操作次数（Input/Output Operations Per Second）

现在普通消费级1TB SSD（尤其是PCIe 3.0/4.0的）顺序读写速度早就跑到5000–7000+ MB/s了，绝大多数人日常使用（游戏、系统、剪辑、办公）都觉得“速度够了”。

但真正能把人卡住、让人觉得“SSD不快”的，往往是随机小文件读写的场景，比如：
- 打开一堆小文件的文件夹
- 游戏加载地图/纹理（大量4K随机读）
- 编译代码、运行虚拟机、数据库查询
- 浏览器开几十个标签快速切换

## 裸盘提交
“裸盘提交” (Raw Disk Submission) 通常指： 在分布式存储、数据库集群（如 Oracle ASM）或虚拟化平台中，用户直接将物理硬盘或未分区逻辑卷交给系统处理，而不经过文件系统（如 NTFS, ext4）格式化。
客户之所以问这个问题，是因为他发现两个关键数值：
•	登记容量：894 (GB) —— 这是硬盘物理硬件标注的原始容量。
•	上报容量：447.130859 (GB) —— 这是系统当前实际识别或使用的容量。
