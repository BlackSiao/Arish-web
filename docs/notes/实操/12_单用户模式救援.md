经常会出现服务器忘记密码或者由于服务异常，导致/tmps/文件被打满，导致无法SSH连接等多种问题
这个时候就必须使用单用户模式来抢进系统里面了。


1. 直接重复服务器，在加载Grub的界面，疯狂的按"↑"和"↓"，跳过倒计时

2. 定位到目标操作系统，按"e"进入系统启动项

3. 找到 intrid=xx quiet , 在后面写上 "quiet init=/bin/bash", 按F10保存后加载

4. 等待系统加载完了之后，不出意外此时就进入了单用户模式，此时的用户也拥有 root 权限

5. 需要设置让硬盘可写，从而能够修改系统文件
```
mount -o remount,rw /
```
