# 单用户模式

有时候经常会出现忘记root密码，或者/tmps/文件夹被打满，导致SSH无法连接的情况

现在可以通过使用单用户模式来抢进虚拟机内部，之后无论是改root密码还是给 /tmps/ 腾空间都没问题

# 步骤

1. 重启Linux设备，在快到GRUB的时候，快速输入 "↑"和"↓"来为接下来的操作争取时间

2. 选择对应的操作系统，按"E"编辑启动项， 在"quiet"后面输入"init /bin/bash/"后， 按 "F10",保存配置后启动

3. 之后就可以进入 单用户系统，有root权限可以进行修改

4. 如果要修改系统文件的话，需要先让系统的磁盘可写: 
```
mount -o remount,rw /
```

5. 此时就算有了系统的最高权限，即使是修改root密码也没有问题
