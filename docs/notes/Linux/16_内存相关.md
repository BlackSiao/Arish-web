## Swap

Swap 就是常说的虚拟内存，在正常情况下服务器运行程序全靠 内存， 但内存被占满的时候，就会触发 OOM-killer 机制，按照内核的
Killer 算法，对全部运行的进程打分，然后Kill掉分最低的。

在有Swap的情况下，Linux内核会将内存中哪些 最不活跃的数据(最少被CPU调用的数据页)， 从内存换到 /swapfile 硬盘空间里面，空出来的内存就可以匀给紧急的程序用。

### 什么时候不推荐（甚至严禁）开 Swap？
很多人会听说“服务器要关闭 Swap 性能才最好”，这也没错，以下场景通常会主动关闭它：
1.
Kubernetes (K8s) 容器集群节点
  K8s 官方在很长一段时间里都强制要求关闭 Swap。因为 K8s 的核心理念是“精确调度”。它需要明确知道每个节点还有多少准确的物理内存。如果开了 Swap，会导致调度器对资源的预估失准。
2.
极致追求低延迟、高并发的纯内存数据库

  例如 
  Redis、ClickHouse、Elasticsearch。这类业务对响应时间是微秒级的。如果开了 Swap，内核一旦把核心数据交换到了“相对极慢”的磁盘上，当程序去读这个数据时，就会发生严重的卡顿（Page Fault / 页缺失），导致整个高并发服务雪崩。

3.
高频核心计算/HPC/量化交易

  宁愿让它因为内存不足直接报错、让人工介入，也绝不能接受程序在硬盘置换中慢吞吞地运行。

### 配置Swap的流程

```
# 以创建一个 16GiB 的 Swap 文件为例

# 1. 在根目录下创建一个 16GiB 的全零空文件
dd if=/dev/zero of=/swapfile bs=1M count=16384

# 2. 赋予安全权限：仅允许 root 用户读写（权限不对内核会拒绝启用）
chmod 600 /swapfile

# 3. 将该文件格式化为交换分区（Swap）格式
mkswap /swapfile

# 4. 立刻激活并启用该 Swap 文件
swapon /swapfile

# 5. 验证 Swap 是否生效
free -h

# 6. 设置永久生效(可选)
默认的 swapon 只是临时生效，机器重启后会失效。需要将其写入系统挂载配置文件中

echo '/swapfile none swap sw 0 0' >> /etc/fstab
```
